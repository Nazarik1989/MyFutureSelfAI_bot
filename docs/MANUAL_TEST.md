# Первый ручной smoke-test в Telegram

Проверку проводите на локальной SQLite-базе. Не публикуйте `.env`, токены, голосовые сообщения и файл `future_self.db`.

## 1. Подготовка Telegram и окружения

1. Откройте официальный аккаунт `@BotFather` в Telegram и выполните `/newbot`.
   - Ожидается: BotFather запросит отображаемое имя и уникальный username, заканчивающийся на `bot`.
2. Завершите создание и скопируйте выданный HTTP API token локально.
   - Ожидается: токен хранится только у вас и не отправляется в чат, issue или git.
3. В каталоге проекта выполните `Set-ExecutionPolicy -Scope Process Bypass`, затем `.\scripts\setup_windows.ps1`.
   - Ожидается: создана `.venv`, установлены зависимости, создан `.env`, миграция находится на `head`; бот не запускается.
4. Получите OpenRouter API key и заполните в `.env` две строки: `TELEGRAM_BOT_TOKEN` и `AI_API_KEY`. Для первого запуска оставьте `TRANSCRIPTION_PROVIDER=disabled`.
   - Ожидается: значения непустые, без кавычек и пробелов по краям; `.env` остаётся локальным.
5. Выполните `.\.venv\Scripts\python.exe -m future_self.doctor --network`.
   - Ожидается: Python, конфигурация, SDK, БД, миграции, Telegram и OpenRouter text LLM имеют статус `OK`; STT отмечен как `disabled`, пользователям ничего не отправлено.
6. Выполните `.\scripts\run_windows.ps1`.
   - Ожидается: Alembic сообщает актуальный `head`, процесс продолжает работать без traceback. Остановить его можно `Ctrl+C`.

## 2. Онбординг, профиль, цели и рутины

1. Откройте созданного бота и отправьте `/start`.
   - Ожидается: создаётся пользователь и появляется первый из 11 вопросов, а также кнопки навигации.
2. Ответьте на два вопроса, остановите процесс `Ctrl+C`, снова запустите `run_windows.ps1` и отправьте `/start`.
   - Ожидается: бот продолжает с сохранённого шага, уже введённые ответы не потеряны.
3. Нажмите `Назад`.
   - Ожидается: показывается предыдущий вопрос; на первом вопросе переход остаётся на первом.
4. На необязательном вопросе нажмите `Пропустить`.
   - Ожидается: бот переходит дальше и не записывает выдуманный ответ.
5. На обязательном вопросе нажмите `Пропустить`.
   - Ожидается: понятное сообщение о невозможности пропуска, текущий шаг не меняется.
6. Нажмите `Отменить`, затем снова `/start`.
   - Ожидается: диалог завершается без удаления ответов и возобновляется с сохранённого шага.
7. Пройдите оставшиеся вопросы, используя реальный IANA timezone, например `Europe/Moscow`.
   - Ожидается: неизвестная timezone отклоняется; после последнего ответа появляется Vision Profile.
8. Нажмите `Редактировать`, измените хотя бы один ответ и снова дойдите до профиля.
   - Ожидается: вопросы начинаются с первого, итог строится по актуальным ответам без новых фактов.
9. Нажмите `Подтвердить`.
   - Ожидается: профиль сохраняется и бот предлагает 3–5 целей с критериями прогресса.
10. Переименуйте одну цель, при наличии 4–5 предложений удалите или отложите одну, затем подтвердите оставшиеся.
    - Ожидается: новое имя сохраняется; обработанная цель не активируется; для подтверждения остаётся 3–5 целей.
11. Подтвердите цели и затем активируйте предложенные рутины.
    - Ожидается: предложено не больше трёх рутин, у каждой показана минимальная версия; после подтверждения они активны.
12. Отправьте `/profile`.
    - Ожидается: отображается подтверждённое резюме, ценности, идентичность, ограничения и стиль поддержки.
13. Отправьте `/today`.
    - Ожидается: короткое напоминание о будущем, один фокус, максимум три действия и минимальный план на сложный день.

## 3. Текстовый и голосовой inbox

1. Отправьте обычным сообщением: `Нужно в пятницу сделать черновик презентации`.
   - Ожидается: до записи в БД показаны исходный текст, тип, заголовок, следующий шаг и три кнопки.
2. Нажмите `Редактировать`, отправьте исправленный вариант.
   - Ожидается: старый вариант не сохраняется; исправленный текст заново классифицируется и снова требует подтверждения.
3. Нажмите `Сохранить`.
   - Ожидается: появляется подтверждение сохранения, запись доступна в `/inbox`.
4. Быстро нажмите `Сохранить` повторно либо повторите старый callback в тестовом клиенте.
   - Ожидается: второй запрос сообщает, что действие уже обработано; дубль в `/inbox` не появляется.
5. Отправьте другую текстовую мысль и нажмите `Не сохранять`.
   - Ожидается: бот подтверждает отказ, запись не появляется в `/inbox`.
6. При `TRANSCRIPTION_PROVIDER=disabled` запишите короткое Telegram voice.
   - Ожидается: бот сообщает, что распознавание временно не настроено, не скачивает аудио и продолжает принимать текст.
7. Для отдельной проверки STT задайте `TRANSCRIPTION_PROVIDER=openai`, отдельный `TRANSCRIPTION_API_KEY`, официальный `TRANSCRIPTION_BASE_URL`, перезапустите бота и снова отправьте voice.
   - Ожидается: появляется расшифровка, затем классификация и кнопки; запрос не уходит в OpenRouter и автоматического сохранения нет. Нажмите `Редактировать`, исправьте ошибку и сохраните.
   - Ожидается: сохраняется исправленный текст, а не первоначальная расшифровка.
8. Отправьте слишком длинное аудио или неподдерживаемый audio-файл.
   - Ожидается: безопасное понятное сообщение, файл и inbox-запись не сохраняются.
9. Выполните `/inbox`.
   - Ожидается: видны только подтверждённые записи текущего пользователя, максимум десять последних.

## 4. Вечер, перезапуск и изоляция

1. Выполните `/evening` и последовательно ответьте: что получилось, что не получилось, энергия `1–5`, препятствие, изменение на завтра.
   - Ожидается: после пятого ответа рефлексия сохраняется; тон сообщения не осуждающий.
2. Повторите `/today`.
   - Ожидается: план учитывает недавнюю выполненную и пропущенную активность.
3. Остановите и снова запустите процесс, затем проверьте `/profile`, `/inbox` и `/today`.
   - Ожидается: подтверждённые данные сохранились в SQLite; при запуске таблицы не очищаются.
4. Откройте бота со второго Telegram-аккаунта и выполните `/start`, `/profile`, `/inbox`.
   - Ожидается: начинается отдельный онбординг; профиль и inbox первого аккаунта недоступны.
5. Просмотрите консоль после намеренной ошибки сети или неверного тестового ключа, не публикуя вывод.
   - Ожидается: пользователь получает нейтральную ошибку; логи не содержат токен, API key, текст голосового или байты аудио.
